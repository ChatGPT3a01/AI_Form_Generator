✨ 提示詞的黃金結構

精心設計的提示詞，運用多個提示詞工程原則，確保 AI 產出符合格式的結果。

1️⃣ 任務定義

「請為『某主題』生成 N 個問卷題目」

2️⃣ 上下文資訊

題型配比、關鍵字、問卷目的

3️⃣ 輸出格式規範

完整的 JSON 結構範例

4️⃣ 品質要求

題目清楚、選項合理、遵守格式

📝 完整提示詞範例

請為「${data.category}」主題生成 ${data.questionCount} 個問卷題目。

題型配比：${data.questionTypes}
關鍵字：${data.keywords}
問卷目的：${data.purpose}

請以 JSON 格式回傳，格式如下：
{
  "questions": [
    {
      "question": "題目文字",
      "type": "單選" 或 "多選" 或 "文字",
      "options": ["選項1", "選項2", ...] // 文字題可省略此欄位
    }
  ]
}

請確保：
1. 題目清楚且符合主題
2. 選項合理且互斥
3. 題型符合配比要求
4. 嚴格遵守 JSON 格式

💡 重要原則：Show, Don't Tell

❌ 不好的寫法

「請用 JSON 格式，包含 questions 陣列，每個問題有 question、type、options 欄位」

✅ 好的寫法

直接展示完整的 JSON 範例結構，AI 看到範例後能精確複製

展示範例比描述規則更有效！AI 能從範例中學習結構，而不是靠理解文字描述來猜測格式。

🔌 Gemini API 呼叫細節

API URL 格式：

https://generativelanguage.googleapis.com/v1beta/models/{model}:generateContent?key={apiKey}

gemini-2.5-flash

速度快，適合即時回應場景，推薦日常使用

gemini-2.5-pro

品質更高，適合複雜任務，API 額度消耗較快

請求體格式：{contents: [{parts: [{text: "提示詞"}]}]}

🔍 解析 AI 回應的技巧

AI 有時會加上額外的說明文字，需要提取純 JSON：

// AI 回應可能是：
以下是為「顧客滿意度調查」生成的問卷題目：
{"questions": [...]}
希望這些題目能幫助您！

// 使用正則表達式提取：
const jsonMatch = text.match(/\{[\s\S]*\}/);
if (jsonMatch) {
  return JSON.parse(jsonMatch[0]);
}
兩階段處理：正則提取 + JSON 解析 = 穩定的 AI 輸出處理

🚀 提示詞優化技巧

技巧一：提供正面範例

加入一兩個實際的好問題作為示範，few-shot learning

技巧二：明確禁止行為

「不要生成一題多問」「不要使用誘導性語言」

技巧三：調整溫度

temperature 0.3 = 保守穩定，0.9 = 多樣創新

技巧四：迭代優化

生成 10-20 次，觀察結果，修改提示詞，重複

💰 Token 用量與成本控制

什麼是 Token？

約 4 個英文字元或 1-2 個中文字

單次問卷生成

提示詞 300-500 tokens + 輸出 500-1000 tokens ≈ 800-1500

Gemini 免費額度：每分鐘 15 次請求、每天 1500 次，個人使用完全足夠！

節省成本的方法：

  • 批次生成：一次請求生成多份問卷
  • 快取常用提示詞片段
  • 使用較小模型或限制 max_tokens
